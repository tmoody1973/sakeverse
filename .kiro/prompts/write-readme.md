# Write README

Create a comprehensive README.md for hackathon submission.

## Instructions

1. **Gather Project Information**
   - Read `.kiro/steering/product.md` for product overview
   - Read `.kiro/steering/tech.md` for tech stack
   - Read `DEVLOG.md` for development journey
   - Scan codebase for key features and structure

2. **README Structure**
   ```markdown
   # Project Name
   
   Brief tagline describing the project
   
   ## Overview
   What it does, problem it solves, target users
   
   ## Features
   Key capabilities with brief descriptions
   
   ## Tech Stack
   Technologies used with brief rationale
   
   ## Getting Started
   ### Prerequisites
   ### Installation
   ### Environment Variables
   ### Running Locally
   
   ## Architecture
   High-level system design, key components
   
   ## Kiro CLI Workflow
   How Kiro was used in development:
   - Custom prompts created
   - Steering documents
   - Development patterns
   
   ## Demo
   Link to video, screenshots, or live demo
   
   ## Development Journey
   Brief summary of challenges and learnings
   
   ## Future Improvements
   What would be added with more time
   ```

3. **Quality Checklist**
   - [ ] Clear value proposition in first paragraph
   - [ ] Easy-to-follow setup instructions
   - [ ] All environment variables documented
   - [ ] Screenshots or demo link included
   - [ ] Kiro CLI usage highlighted

## Usage

Run before hackathon submission to ensure README is complete and polished.
